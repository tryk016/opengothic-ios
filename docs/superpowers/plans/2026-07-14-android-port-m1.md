# OpenGothic Android Port — M1 (boot-to-menu) Implementation Plan

> **Status: ARCHIVED / COMPLETED.** This plan records the original M1
> implementation. The active status and backlog are in
> [`android/TODO.md`](../../../android/TODO.md); real-device results are in
> [`2026-07-16-android-m2-report.md`](../reports/2026-07-16-android-m2-report.md).
>
> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Boot the shared OpenGothic engine on Android (GameActivity + Vulkan) and render the main menu with basic touch navigation, on the `android` branch, without touching the iOS/desktop targets.

**Architecture:** Add a new `android/` Gradle+NDK project that drives the *existing* top-level `CMakeLists.txt` via `externalNativeBuild`. Add a Tempest `AndroidApi` windowing backend (on `game-activity/native_app_glue`) plus a `VK_KHR_android_surface` swapchain path. The engine runs on the glue's dedicated `android_main` thread (no iOS-style fibers needed — that thread is ours to block). Game data lives in public `/sdcard/OpenGothic/`. CI builds a signed APK and publishes it to a GitHub Release.

**Tech Stack:** Android NDK (r26+), GameActivity (`androidx.games:games-activity`), Gradle 8 / AGP 8, CMake, Vulkan (device-side, via NDK `libvulkan.so`), glslang (host tool, unchanged), Kotlin (one tiny Activity subclass).

## Global Constraints

- Branch: **`android`** only. Never modify or rely on `master`. Push to `origin android` triggers CI.
- **Nothing compiles or runs on Windows.** The CI build is the verification step; the user installs and tests on-device. Never claim "works" before CI is green + device confirms. State clearly that changes are UNVERIFIED until then.
- All Android code MUST be guarded by `#if defined(__ANDROID__)` so the Windows/iOS/Linux builds are byte-for-byte unaffected.
- Tempest engine edits go through **`android/patches/apply-patches.sh`** (idempotent, like `ios/patches/apply-patches.sh`). The `lib/Tempest` submodule is intentionally dirty — never `git add` inside it.
- ABI: **`arm64-v8a` only.**
- Target device: Mediatek Helio G99 / Mali-G57 MC2, Vulkan 1.1+, 4 GB RAM, Android (arm64).
- Data root: **`/sdcard/OpenGothic/`** (game data under `Gothic2/`; `Gothic.ini` and saves alongside). Permission: `MANAGE_EXTERNAL_STORAGE`.
- Signing: **committed keystore** `android/keystore/opengothic.jks` (alias `opengothic`, storepass/keypass `opengothic` — a throwaway key for a personal sideload port). Stable signature → in-place updates.
- Package / lib names (must stay consistent everywhere): applicationId **`opengothic.gothic2`**, native lib **`Gothic2Notr`** (GameActivity loads `libGothic2Notr.so`), manifest `android.app.lib_name` = `Gothic2Notr`.
- Commit style: `feat(android): …`, `fix(android): …`, `docs(android): …`, `build(android): …`.
- Keep `android/TODO.md` updated as the running status list.

- [ ] **Step 7: Write the `AndroidApi` backend skeleton**

`apply-patches.sh` creates `lib/Tempest/Engine/system/api/androidapi.h` mirroring `iosapi.h`:

```cpp
#pragma once
#include <Tempest/SystemApi>
struct android_app;
struct ANativeWindow;

namespace Tempest {
class AndroidApi final: SystemApi {
  public:
    using SystemApi::dispatchRender;
    static void  setAndroidApp(android_app* app);
  private:
    AndroidApi();
    Window*  implCreateWindow(Tempest::Window* owner, uint32_t width, uint32_t height) override;
    Window*  implCreateWindow(Tempest::Window* owner, ShowMode sm) override;
    void     implDestroyWindow(Window* w) override;
    void     implExit() override;
    Rect     implWindowClientRect(SystemApi::Window* w) override;
    bool     implSetAsFullscreen(SystemApi::Window* w, bool fullScreen) override;
    bool     implIsFullscreen(SystemApi::Window* w) override;
    void     implSetCursorPosition(SystemApi::Window* w, int x, int y) override;
    void     implShowCursor(SystemApi::Window* w, CursorShape cursor) override;
    bool     implIsRunning() override;
    int      implExec(AppCallBack& cb) override;
    void     implProcessEvents(AppCallBack& cb) override;
    void     implSetWindowTitle(SystemApi::Window* w, const char* utf8) override;
  friend class SystemApi;
  };
}
```

`androidapi.cpp` implements it against `game-activity/native_app_glue`. **This is the hardest part of M1 — it will take CI iteration.** The contract to implement:

- `setAndroidApp(android_app*)` stores the glue app pointer (set from `android_main` before `Application::exec`).
- The `SystemApi::Window*` handle returned by `implCreateWindow` IS the `ANativeWindow*` from `app->window`. `implWindowClientRect` returns `ANativeWindow_getWidth/Height`.
- `implProcessEvents(cb)` drains the glue's event + input queues once (non-blocking): poll `ALooper_pollAll`, process `app->cmd` sources (`APP_CMD_INIT_WINDOW` → window ready, `APP_CMD_TERM_WINDOW` → surface lost, `APP_CMD_WINDOW_RESIZED`), and pump `android_input_buffer` motion events (→ `SystemApi::dispatchMouseDown/Up/Move` for Task 4). Then call `dispatchRender` on the active window.
- `implExec(cb)` loops `while(isRunning()) implProcessEvents(cb);`.
- Use the iOS backend (`iosapi.mm`) as the structural reference for how the impl* methods map onto `SystemApi::Window*`, event dispatch, and render pumping.

For **Task 1 only**, `implProcessEvents` may skip input pumping (added in Task 4) — it just needs cmd handling + render.
